## 🎯 변경 이유 및 장점

### 1. QUICKSTART → docs/00_빠른시작가이드.md
**이유**: 
- 문서 번호 체계 통일 (00번대 = 필독/시작 문서)
- 문서 목록에서 쉽게 찾을 수 있음

**장점**:
- 논리적 순서: 00_빠른시작 → 00_기술스택 → 01_요구사항 → ...
- 일관성: 모든 주요 문서가 docs/ 하위에 위치

### 2. 테스트 도구 → scripts/
**이유**:
- `scripts/`는 실행 가능한 스크립트 전용 디렉토리
- `install-jdk.sh`, `install-jboss.sh`와 같은 레벨에 배치

**장점**:
- 명확한 분리: 문서(docs/) vs 스크립트(scripts/)
- 실행 도구가 한 곳에 모임
- `./scripts/test-api.sh` - 직관적인 경로

### 3. 문서 번호 10 → 08
**이유**:
- 10번과 99번이 멀리 떨어져 있어 혼란
- 08번은 논리적으로 "설계(01~05) → 분석(06) → 배포(08) → 설치(99)"

**장점**:
- 순차적 흐름: 설계 → 개발 → 배포 → 환경 설정
- 번호 공백(07, 09)은 향후 확장 여지

### 4. 10_문서요약.md 삭제
**이유**:
- 00_문서목록_최종.md와 내용 중복
- 동일한 정보를 두 곳에서 관리할 필요 없음

**장점**:
- 단일 진실 공급원 (Single Source of Truth)
- 유지보수 부담 감소
